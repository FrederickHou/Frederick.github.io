并查集介绍

并查集（Union-find Sets）是一种非常精巧而实用的数据结构，它主要用于处理一些不相交集合的合并问题。一些常见的用途有求连通子图、求最小生成树的 Kruskal 算法和求最近公共祖先（Least Common Ancestors, LCA）等。应用场景：比如说编译器判断同一个变量的不同引用，比如社交网络中的朋友圈计算等等。

关键字

  • 合并（Union)： 把两个不相交的集合合并为一个集合。
  • 查询（Find): 查询两个元素是否在同一个集合中。

并查集还有两种优化：

  • 路径压缩: 就是在每次查找时，令查找路径上的每个节点都直接指向根节点，因此O(1) 的时间找到某一节点的根节点。
  • 按秩合并: 使用秩来表示树高度的上界，在合并时，总是将具有较小秩的树根指向具有较大秩的树根。简单的说，就是总是将比较矮的树作为子树，添加到较高的树中。(解决树的平衡性问题)

示例

班上有 N 名学生。其中有些人是朋友，有些则不是。他们的友谊具有是传递性。如果已知 A 是 B 的朋友，B 是 C 的朋友，那么我们可以认为 A 也是 C 的朋友。所谓的朋友圈，是指所有朋友的集合。

给定一个 N * N 的矩阵 M，表示班级中学生之间的朋友关系。如果M[i][j] = 1，表示已知第 i 个和 j 个学生互为朋友关系，否则为不知道。你必须输出所有学生中的已知的朋友圈总数。

输入: 
[[1,1,0],
[1,1,0],
[0,0,1]]
输出: 2 
说明：已知学生0和学生1互为朋友，他们在一个朋友圈。
第2个学生自己在一个朋友圈。所以返回2。

思路1(图的邻接矩阵，进行深度优先遍历)：

给定点与点关系的矩阵，其实就是无向的图进行邻接矩阵进行存储。

对图的每个点进行深度优先遍历，并且对访问过的点进行标记保存访问记录vistied[i]。

每一次从新的点进行深度优先遍历，且每个节点只访问过一次，每次得到的结果就是一个并集的朋友圈。

最后结果就是有几个没有被访问过的点，就有朋友圈存在。
