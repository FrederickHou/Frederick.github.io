背包问题

题目(01背包)

在M件物品取出若干件放在空间为M的背包里，每件物品的体积为W1，W2……Wn，与之相对应的价值为P1,P2……Pn。求出获得最大价值的方案。

注意 ：在本题中，所有的体积值均为整数。此题属于01背包问题，01的意思是，每个物品都是一个整体，要么整个都要，要么都不要。

思路

分析：对于一种物品，要么装入背包，要么不装。所以对于一种物品的装入状态可以取0和1。设物品i的装入状态为xi,xi∈ (0,1)，此问题称为0-1背包问题。

定义dp[i][j]为前i个物品，在背包容量是j时能够装入的最大价值。

如果当前背包容量大于物品的质量便可装入，反之不能装入，背包当前价值维持前i-1个物品，在背包容量是j时的最大价值。

如果可以装入，那么要和当前第i个物品的价值+ (前i-1个商品中，当前背包总量-当前物品的重量的物品所对应的最大价值) 和前i-1个商品中重量是j时得到的最大价值做比较。

最后得到的dp[M][W]就是背包容量为W时前M个物品所能装载的最大价值。

题解

题目-零钱兑换（完全背包）

给定不同面额的硬币和一个总金额。写出函数来计算可以凑成总金额的硬币组合数。假设每一种面额的硬币有无限个。

示例 1:

输入: amount = 5, coins = [1, 2, 5]
输出: 4
解释: 有四种方式可以凑成总金额:
5=5
5=2+2+1
5=2+1+1+1
5=1+1+1+1+1

示例 2: 

输入: amount = 3, coins = [2]
输出: 0
解释: 只用面额2的硬币不能凑成总金额3。

示例 3:

输入: amount = 10, coins = [10] 
输出: 1

思路

我们可以把这个问题转化为背包问题的描述形式：有一个背包，最大容量为amount，有一系列物品coins，每个物品的重量为coins[i]，每个物品的数量无限。请问有多少种方法?每个物品的数量是无限的，这也就是传说中的「完全背包问题」。

确定dp数组的定义。首先看看刚才找到的「状态」，有两个，也就是说我们需要一个二维dp数组。dp[i][j]的定义如下：若只使用前i个物品，当背包容量为j时，有dp[i][j]种方法可以装满背包。换句话说，翻译回我们题目的意思就是：若只使用coins中的前i个硬币的面值，若想凑出金额j，有dp[i][j]种凑法。经过以上的定义，可以得到：base case 为dp[0][…] = 0， dp[…][0] = 1。因为如果不使用任何硬币面值，就无法凑出任何金额；如果凑出的目标金额为 0，那么“无为而治”就是唯一的一种凑法。我们最终想得到的答案就是dp[N][amount]，其中N为coins数组的大小。

根据「选择」，思考状态转移的逻辑。注意，我们这个问题的特殊点在于物品的数量是无限的，所以这里和之前写的背包问题文章有所不同。如果你不把这第i个物品装入背包，也就是说你不使用coins[i]这个面值的硬币，那么凑出面额j的方法数dp[i][j]应该等于dp[i-1][j]，继承之前的结果。如果你把这第i个物品装入了背包，也就是说你使用coins[i]这个面值的硬币，那么dp[i][j]应该等于dp[i][j-coins[i-1]]。首先由于i是从 1 开始的，所以coins的索引是i-1时表示第i个硬币的面值。dp[i][j-coins[i-1]]也不难理解，如果你决定使用这个面值的硬币，那么就应该关注如何凑出金额j - coins[i-1]。比如说，你想用面值为 2 的硬币凑出金额 5，那么如果你知道了凑出金额 3 的方法，再加上一枚面额为 2 的硬币，就可以凑出 5 了。

题解

优化

dp数组的转移只和dp[i][…]和dp[i-1][…]有关，所以可以压缩状态，进一步降低算法的空间复杂度。
